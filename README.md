## ENGINEERING DOCTRINE

### 01. Auditability over magic

AI is useful only when its actions can be explained, traced, constrained, and reviewed.

### 02. Determinism before autonomy

Autonomous systems must first obey rules, boundaries, permissions, and evidence.

### 03. Infrastructure before hype

The strongest systems are boring at the core and beautiful at the interface.

### 04. Interfaces must reveal state

A good UI does not only look clean.  
It tells the operator what is true, what is risky, and what must happen next.

### 05. Peru is not a small market problem

Peru is a complex compliance laboratory.  
If a system survives SUNAT, multi-RUC workflows, informal operations, and fragmented data, it can evolve into LATAM-grade infrastructure.
